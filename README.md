## ローカル開発環境のセットアップ

### 必要なもの

- Node.js v20 以上
- Docker Desktop（PostgreSQL 用）
- Google アカウント（OAuth 認証情報の取得用）

---

### 手順

#### 1. 依存関係のインストール

```bash
npm install
```

#### 2. 環境変数の設定

```bash
cp .env.example .env.local
```

`.env.local` を開き、各変数を設定します（後述）。

#### 3. PostgreSQL を起動

##### 方法A: Docker（Docker Desktop がある場合）

```bash
docker compose up -d
```

起動確認：
```bash
docker compose ps
# postgres が "healthy" になっていることを確認
```

##### 方法B: Homebrew のネイティブ PostgreSQL（Docker を使わない場合）

Docker が無い環境では Homebrew の PostgreSQL でも代用できます。

```bash
# PostgreSQL 16 をインストール（未導入の場合）
brew install postgresql@16

# サービスとして起動（再起動後も自動起動）
brew services start postgresql@16

# データベースを作成
createdb baseball_match

# .env.local の DSN（postgres:postgres）に合わせてロールを作成
psql -d baseball_match -c "CREATE ROLE postgres LOGIN SUPERUSER PASSWORD 'postgres';"
```

起動確認：
```bash
pg_isready -h localhost -p 5432
# "accepting connections" と表示されれば OK
```

#### 4. Google OAuth 認証情報の取得

1. [Google Cloud Console](https://console.cloud.google.com/apis/credentials) を開く
2. 「認証情報を作成」→「OAuth 2.0 クライアント ID」を選択
3. アプリケーションの種類: **ウェブアプリケーション**
4. 承認済みのリダイレクト URI に追加:
   ```
   http://localhost:3000/api/auth/callback/google
   ```
5. 作成後に表示される **クライアントID** と **クライアントシークレット** をメモ

#### 5. `.env.local` の設定

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/baseball_match"

AUTH_SECRET="任意の文字列（openssl rand -base64 32 で生成推奨）"
AUTH_GOOGLE_ID="手順4で取得したクライアントID"
AUTH_GOOGLE_SECRET="手順4で取得したクライアントシークレット"
```

`AUTH_SECRET` の生成コマンド（ターミナルで実行）：
```bash
openssl rand -base64 32
```

#### 6. データベースのマイグレーション

> **注意:** Prisma CLI は `.env` は自動で読み込みますが、Next.js 用の `.env.local` は読み込みません。
> そのため `.env.local` を明示的に読み込んでから実行します（読み込まないと
> `Environment variable not found: DATABASE_URL` エラーになります）。

```bash
set -a && source .env.local && set +a && npx prisma db push
```

> 毎回入力したくない場合は、`.env` を `.env.local` へのシンボリックリンクにしておくと
> 以降は `npx prisma db push` だけで動きます（`prisma studio` など他コマンドも同様）：
> ```bash
> ln -s .env.local .env
> ```

成功すると以下のような出力が表示されます：
```
✓ Generated Prisma Client
The database is now in sync with your Prisma schema.
```

#### 7. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

## 動作確認手順

### 1. Googleログイン

1. `http://localhost:3000` を開く
2. 「Googleでログイン」ボタンをクリック
3. Google アカウントを選択してログイン
4. `/dashboard` にリダイレクトされることを確認

**確認ポイント：**
- ランディングページにアプリ名とログインボタンが表示される
- ログイン後にダッシュボードに遷移する
- 未ログインで `/dashboard` に直接アクセスすると `/` にリダイレクトされる
- ログイン済みで `/` にアクセスすると `/dashboard` にリダイレクトされる

---

### 2. チーム作成

1. ダッシュボードの「+ チームを作成」ボタンをクリック
2. チーム名を入力（例: 渋谷ベアーズ）
3. 説明を入力（任意）
4. 「チームを作成する」をクリック
5. ダッシュボードに戻り、作成したチームが表示されることを確認

**確認ポイント：**
- チーム名が空のまま送信するとエラーが表示される（「チーム名を入力してください」）
- 51文字以上のチーム名を入力するとエラーが表示される
- 作成成功後にダッシュボードへリダイレクトされる
- 複数チームを作成して、すべてが一覧に表示されることを確認する

---

### 3. チーム一覧表示

**確認ポイント：**
- 作成したチームが一覧に表示される
- チームカードに以下が表示される：
  - チーム名
  - 「管理者」バッジ（自分が作成したチームは管理者）
  - 説明文（入力した場合）
  - メンバー数（「メンバー 1人」）
  - 6文字の招待コード（`A3K9ZR` のような形式）
- 招待コードの「コピー」をクリックするとクリップボードにコピーされ「コピー済み ✓」に変わる
- チームがない状態ではエンプティステートが表示される

---

### 4. チーム詳細・招待コードの確認

ダッシュボードのチームカード（チーム名）をタップすると `/teams/[teamId]` のチーム詳細画面に遷移します。

- [ ] チームカードをタップするとチーム詳細画面が開く
- [ ] 説明文・メンバー一覧（名前・アイコン・役割バッジ）が表示される
- [ ] 自分の行に「（あなた）」が表示される
- [ ] **管理者の場合のみ** 招待コードが表示される
- [ ] 招待コードの「コピー」をタップするとコピーされ「コピー済み ✓」に変わる
- [ ] **管理者でない（PLAYER）メンバー**には招待コードが表示されない
- [ ] 所属していないチームの `/teams/[teamId]` に直接アクセスすると 404 になる
- [ ] 存在しない teamId にアクセスすると 404 になる

---

### 5. チーム参加（招待コード）

別の Google アカウントでログインし直すと、参加フローを確認しやすいです。

#### 参加の成功

1. ダッシュボード右上の「参加」ボタン（またはエンプティステートの「招待コードで参加する」）をタップ
2. `/teams/join` の招待コード入力画面が開く
3. 管理者から共有された6文字の招待コードを入力（小文字で入力しても自動で大文字になる）
4. 「チームに参加する」をタップ

- [ ] 「参加」ボタンから招待コード入力画面に遷移する
- [ ] 入力欄が大文字・等幅フォントで表示される
- [ ] 正しい招待コードで参加すると、そのチームの詳細画面 `/teams/[teamId]` にリダイレクトされる
- [ ] 参加後、ダッシュボードの所属チーム一覧にそのチームが追加される
- [ ] 参加したメンバーの役割は「メンバー」（PLAYER）になっている

#### エラーハンドリング

- [ ] 空のまま送信すると「招待コードを入力してください」が表示される
- [ ] 存在しない招待コードを入力すると「招待コードが正しくありません」が表示される
- [ ] **すでに参加済み**のチームの招待コードを入力すると「すでにこのチームに参加しています」が表示される
- [ ] エラー時は画面が遷移せず、入力した内容が残る

#### API 単体での確認（オプション）

ログイン済みのブラウザの DevTools コンソールから直接 API を叩いて確認できます：

```js
await fetch("/api/teams/join", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ inviteCode: "ABC123" }),
}).then((r) => r.json())
```

- [ ] 正しいコード → `201` と `{ ok: true, teamId }`
- [ ] 存在しないコード → `404` と `{ error: "招待コードが正しくありません" }`
- [ ] 参加済み → `409` と `{ error: "すでにこのチームに参加しています", teamId }`
- [ ] 未ログイン状態 → `401` と `{ error: "ログインが必要です" }`

### 6. 試合作成・試合一覧

#### 試合の作成（管理者）

1. **管理者**としてチーム詳細画面 `/teams/[teamId]` を開く
2. 「試合」セクションの「+ 試合を作成」をタップ → `/teams/[teamId]/games/new`
3. 試合日時・場所・集合時間・試合開始時間を入力、定員/メモは任意 → 「試合を作成する」

- [ ] 管理者には「+ 試合を作成」ボタンが表示される
- [ ] 必須項目（日時・場所・集合時間・試合開始時間）が空だとエラーが表示され送信されない
- [ ] 作成成功で試合詳細 `/teams/[teamId]/games/[gameId]` にリダイレクトされる
- [ ] 作成した試合がチーム詳細の「今後の試合」に表示される

#### アクセス制御

- [ ] **管理者でないメンバー**には「+ 試合を作成」ボタンが表示されない
- [ ] 管理者でないメンバーが `/teams/[teamId]/games/new` に直接アクセスすると
      チーム詳細にリダイレクトされる（作成画面が出ない）
- [ ] 所属していないチームの試合詳細 URL は 404 になる
- [ ] 別チームの gameId を URL に入れても 404 になる

#### 一覧表示（今後／過去）

- [ ] チーム詳細で試合が「今後の試合」「過去の試合」に分かれて表示される
      （過去は直近が上、薄く表示）
- [ ] 試合が1件も無いときはエンプティステートが表示される
- [ ] 各試合カードに「開始 {試合開始時間} ・ 定員 {人数} ・ 参加 {人数}」が表示される
      （集合時間ではなく試合開始時間が表示される）
- [ ] 出欠の参加人数が、回答状況に応じてリアルタイムに変わる
      （出欠回答後にチーム詳細に戻ると人数が更新されている）
- [ ] 各試合をタップすると試合詳細が開き、日時・試合開始時間・集合時間・場所・定員・メモが表示される

#### API 単体での確認（オプション）

ログイン済みブラウザの DevTools コンソールから（`TEAM_ID` は実在の管理チームIDに置換）：

```js
// 作成（管理者）
await fetch("/api/teams/TEAM_ID/games", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    startsAt: new Date("2026-07-01T10:00").toISOString(),
    location: "区民グラウンド",
    meetTime: "09:30",
    startTime: "10:00",
    capacity: 15,
    note: "スパイク不可",
  }),
}).then((r) => r.json())

// 一覧（メンバー）
await fetch("/api/teams/TEAM_ID/games").then((r) => r.json())
```

- [ ] 管理者が正しい入力で作成 → `201` と `{ ok: true, gameId }`
- [ ] 入力不備（例: `meetTime: "25:00"` や `startTime: "25:00"`）→ `400` とエラーメッセージ
- [ ] 管理者でないメンバーが作成 → `403`（`試合を作成できるのは管理者のみです`）
- [ ] メンバーでないユーザーが一覧取得 → `403`
- [ ] 未ログインで作成/一覧 → `401`

### 7. 試合の出欠管理

試合詳細画面 `/teams/[teamId]/games/[gameId]` で、選手は自分の出欠を回答し、
管理者は全員の出欠状況を確認できる。

#### 選手側：出欠の回答（決定ボタン＋楽観的更新）

1. チームメンバーとして試合詳細画面を開く
2. 「あなたの出欠」の **参加 ✅ / 欠席 ❌ / 未定 ❓** いずれかをタップ（この時点ではまだ未確定）
3. 「決定」ボタンをタップして送信する

- [ ] ボタンをタップした時点では選択状態（青）になるだけで、まだAPIには送信されない
- [ ] 選択直後は「決定」ボタンが有効（青）になり、「『決定』を押すと回答が保存されます」と表示される
- [ ] 確定済みの回答と同じボタンを選んでいる間は「決定」ボタンが無効（グレー・押せない）
- [ ] 「決定」を押した瞬間に確定状態へ切り替わる（= 楽観的更新。API 応答を待たずに見た目が変わる）
- [ ] リロードしても直前に決定した回答が選択状態のまま残る（保存されている）
- [ ] 確定後に別のボタンを選び直すと「決定」が再度有効になり、押すと回答が切り替わる（いつでも変更可能）
- [ ] 未回答のときは「まだ回答していません」と表示される
- [ ] （失敗時）オフライン等で保存に失敗すると確定状態は変わらず、エラーメッセージが出る
      （DevTools の Network を Offline にして決定ボタンを押して確認）

#### 管理者側：出欠状況の確認

1. **管理者**として同じ試合詳細画面を開く

- [ ] 「出欠状況」に **参加 / 欠席 / 未定 / 未回答** の人数が表示される
- [ ] 人数の合計がチームの総メンバー数と一致する（未回答 = 総数 − 回答済み）
- [ ] 「メンバー別の回答」に、回答ごとにグループ分けされたメンバー名の一覧が表示される
- [ ] 誰かが回答を変えると、人数と名簿に反映される（回答後に画面が更新される）
- [ ] **管理者でないメンバー**には「メンバー別の回答」名簿は表示されない（人数サマリーは表示される）

#### API 単体での確認（オプション）

ログイン済みブラウザの DevTools コンソールから（`GAME_ID` は実在の試合IDに置換）：

```js
// 出欠の登録・更新（チームメンバー）
await fetch("/api/games/GAME_ID/attendance", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ status: "ATTENDING" }), // ATTENDING | ABSENT | UNDECIDED
}).then((r) => r.json())
```

- [ ] メンバーが正しい値で送信 → `200` と `{ ok: true, status: "ATTENDING" }`
- [ ] 同じ試合に再送信すると回答が上書きされる（重複行は作られない / `upsert`）
- [ ] 不正な値（例: `status: "MAYBE"`）→ `400`（`出欠の値が不正です`）
- [ ] **チームメンバーでない**ユーザーが送信 → `403`（`このチームのメンバーではありません`）
- [ ] 存在しない `gameId` → `404`
- [ ] 未ログインで送信 → `401`

### 8. 助っ人募集

試合詳細画面で管理者が助っ人を募集し、`/helpers` でチーム外のユーザーも応募できる。

#### 管理者側：募集の作成・応募者確認・締め切り

1. **管理者**として試合詳細画面 `/teams/[teamId]/games/[gameId]` を開く
2. 「助っ人募集」セクションの「+ 募集を作成」をタップ
3. ポジション（任意）・募集人数（必須）・メモ（任意）を入力 → 「募集を作成する」

- [ ] 「+ 募集を作成」は**管理者にのみ**表示される（「助っ人募集」セクション自体が管理者専用）
- [ ] 募集人数が空のまま送信するとエラーが表示され送信されない
- [ ] 作成した募集が一覧に「募集中」バッジ付きで表示される
- [ ] 募集カードに「応募者一覧」が表示される（未応募時は「まだ応募はありません」）
- [ ] 助っ人側で応募すると、リロード後に応募者一覧に名前が追加される
- [ ] 「募集を締め切る」をタップすると、バッジが「募集終了」に変わり締め切りボタンが消える
- [ ] 締め切り後の募集は `/helpers` でも「募集終了」と表示され、応募できなくなる

#### 助っ人側：公開一覧での応募

1. ログアウト状態（または別アカウント・チーム外ユーザー）で `/helpers` を開く
   （トップページ `/` の「助っ人募集を見る」リンクからも遷移可能）

- [ ] **未ログインでも** `/helpers` の募集一覧が閲覧できる（試合日時・場所・開始時間・チーム名・募集人数・応募人数が表示される）
- [ ] 未ログインのカードには「ログインして応募する」リンクが表示され、応募ボタンは押せない
- [ ] ログイン後（チーム外ユーザー）は「応募する」ボタンが表示される
- [ ] 「応募する」をタップした瞬間に「応募済み ✓」に変わる（楽観的更新。API応答を待たない）
- [ ] リロードしても「応募済み ✓」のまま残る（保存されている）
- [ ] 同じ募集に再度応募してもエラーにならない（二重応募にならない）
- [ ] **募集元チームのメンバー**（管理者・一般メンバーどちらも）には応募ボタンが表示されない
      （出欠で参加を回答すればよいため）
- [ ] 募集が無いとき・全て募集終了のときはエンプティステート/「募集終了」表示になる

#### API 単体での確認（オプション）

ログイン済みブラウザの DevTools コンソールから（`GAME_ID`・`REQUEST_ID` は実在のIDに置換）：

```js
// 募集作成（管理者）
await fetch("/api/games/GAME_ID/helper-requests", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ positions: "外野手", capacity: 2, note: "経験者優先" }),
}).then((r) => r.json())

// 応募（チーム外ユーザー）
await fetch(`/api/games/GAME_ID/helper-requests/REQUEST_ID/apply`, {
  method: "POST",
}).then((r) => r.json())

// 締め切り（管理者）
await fetch(`/api/games/GAME_ID/helper-requests/REQUEST_ID`, {
  method: "PATCH",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ status: "CLOSED" }),
}).then((r) => r.json())
```

- [ ] 管理者が正しい入力で募集作成 → `201` と `{ ok: true, requestId }`
- [ ] 管理者でないメンバーが募集作成 → `403`（`助っ人募集を作成できるのは管理者のみです`）
- [ ] 募集人数が範囲外（例: `capacity: 0`）→ `400`
- [ ] チーム外ユーザーが応募 → `200` と `{ ok: true, applied: true }`
- [ ] **募集元チームのメンバー**が応募 → `403`（`チームメンバーは応募できません`）
- [ ] 締め切り済みの募集に応募 → `400`（`この募集は終了しています`）
- [ ] 管理者が締め切り → `200` と `{ ok: true, status: "CLOSED" }`
- [ ] 管理者でないユーザーが締め切りを実行 → `403`
- [ ] 未ログインで応募/作成/締め切り → `401`

- [ ] **試合作成・一覧**：管理者が試合（日時・場所・集合時間・試合開始時間・定員）を作成でき、
      今後／過去に分けて一覧表示される
- [ ] **出欠管理**：メンバーが参加/欠席/未定を回答・変更でき、管理者は人数サマリーと名簿を確認できる
- [ ] **助っ人募集（管理者）**：試合ごとに募集を作成し、応募者一覧を確認し、募集を締め切れる
- [ ] **助っ人募集（チーム外ユーザー）**：`/helpers` で募集を閲覧し、ログインして応募できる。
      応募後は「応募済み」と表示され、チームメンバーには応募ボタンが表示されない
- [ ] **アクセス制御の横断確認**：
  - 所属していないチーム／別チームの試合 URL への直接アクセスは 404
  - 管理者専用の操作（試合作成・出欠の名簿表示・募集作成・締め切り）は非管理者には表示/実行できない
  - 各 API は未ログイン → `401`、権限不足 → `403` を返す

## Vercel へのデプロイ（本番環境）

1. [Vercel](https://vercel.com) でプロジェクトを作成してリポジトリを接続
2. Vercel Postgres を作成してプロジェクトに接続（`DATABASE_URL` が自動設定される）
3. 以下の環境変数を Vercel ダッシュボードで設定：
   - `AUTH_SECRET`
   - `AUTH_GOOGLE_ID`
   - `AUTH_GOOGLE_SECRET`
4. Google Cloud Console でリダイレクト URI を追加：
   ```
   https://your-domain.vercel.app/api/auth/callback/google
   ```
5. デプロイ後のターミナルで以下を実行してスキーマを適用：
   ```bash
   npx prisma db push
   ```
